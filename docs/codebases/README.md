# docs/codebases/ — one doc per app

One file per app in [apps.yaml](../../apps.yaml), named after its `apps/` folder
(`backend.md`, `frontend.md`, …). Created at bootstrap from [TEMPLATE.md](TEMPLATE.md);
the `/new-app` skill creates one for each new app.

Each doc answers, briefly: what the app is, its tech stack, how the code is organized
(links, not prose restating structure), and a **"how to add a feature" checklist** with one
exemplar to copy per step. Link to code as `../../apps/<app>/...`.
